1. Создайте класс объявляемого исключения InsufficientFundsException. Это исключение должно выбрасываться при попытке списать со счета сумму, превышающую остаток или лимит по кредитной карте.
2. Переопределите метод equals(Object obj) и hashCode() в классах DebitAccount и CreditAccount. Метод equals(Object obj) должен возвращать истину, только если сравниваемый объект obj является экземпляром того же класса, и значения полей номер карты равны.
Стандартная практика реализации метода hashCode():
	Взять некоторое достаточно большое простое число
	Выполнить операцию исключающее или (^) между этим числом и значениями всех полей класса.
		Если поле x – примитив (boolean, byte, short, int, char) – берется само значение.
		Если поле x типа long, float, double, то берется результат метода hasCode соответствующего класса обертки (т.е. Long.hashCode(x), Float.hashCode(x), Double.hashCode(x)).
		Если поле x ссылочного типа, берется результат вызова его метода x.hashCode().
   
3. Переопределите метод toString() во всех классах пакета. Метод должен возвращать строку, содержащую полное описание класса, со значениями всех его полей. При реализации метода использовать класс StringBuilder.

4. в файлах описания DebitAccount и CreditAccount, требуется указать только то, что подвергнется изменениям.

Ошибка: InsufficientFundsException{message=Превышение кредитного лимита. Доступно: 7090.0} Почему «Превышение кредитного лимита»?
Потому что при попытке снять 10000 остаток на счёте станет 2090 - 10000 = -7910, что ниже установленного лимита -5000. Это превышает допустимый долг, поэтому выбрасывается исключение с соответствующим текстом.
Почему «Доступно: 7090.0»?
Значение balance - limit показывает максимальную сумму, которую можно снять без превышения лимита:
	2090 - (-5000) = 2090 + 5000 = 7090.
Именно столько клиент может снять в данный момент (учитывая собственные 2090 и кредитные 5000). Запрошено 10000 — больше допустимого.
