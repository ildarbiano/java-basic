✅ Шаг 1. Переключиться на целевую ветку
В IntelliJ:
👉 правый нижний угол → выбрать main или через терминал:
git checkout main


✅ Шаг 2. Обновить main (если нужно)
Если удалённая ветка могла измениться:
git pull

✅ Шаг 3. Влить ветку
git merge feature/login
Если всё хорошо → увидишь сообщение про успешный merge.
⚠️ Если будут конфликты
IntelliJ откроет окно Merge:
Accept Yours
Accept Theirs
Merge вручную
После исправления:
git add .
git commit
(если commit не создался автоматически)

✅ Шаг 4. Запушить в удалённый репозиторий
git push
Готово 🚀

========== через транзитную ветку ===========================
1. Встать на <транзитную ветку>
2. cmd:
git rebase <целевая ветка>
2. Конфликты: появится окно со списком конфликтов
3. Список конфликтов:
git status
4. Разобрать конфликты, все
5. Создать ветку <транзитную ветку+fix>, на основе <транзитную ветку> 
Git/New Branch
6. Сделать PullRequest в GUI remote:
<транзитную ветку+fix>
<целевая ветка>
7. Merge PullRequest в GUI remote, с удалением <транзитную ветку+fix>
