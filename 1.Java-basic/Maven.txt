================== MANIFEST pom.xml JAR ==============================================
MANIFEST.MF лежит, В jar: META-INF/MANIFEST.MF
- ключевой компонент JAR-файла, который содержит метаданные, необходимые для правильного запуска приложения. Этот файл определяет точку входа в приложение, версию, зависимости и другую важную информацию. говорит JVM, что запускать. Некоторые области применения MANIFEST.MF:
	-Создание исполняемых JAR-приложений — определение основного класса для запуска приложения;
	-Управление зависимостями — указание внешних библиотек;
	-Конфигурация приложения — предоставление информации во время выполнения;
	-Безопасность и подписание — включение метаданных, связанных с безопасностью.
MANIFEST описывается ТОЛЬКО через pom.xml
MAVEN читает MANIFEST описанный через pom.xml всё, что должно попасть в JAR → pom.xml
maven-jar-plugin 		плагин в pom.xml создают описание манифеста для запуска архива приложения.
maven-shade-plugin		плагин в pom.xml создают описание манифеста для запуска архива приложения.

============= плагин maven-jar-plugin ===========================
учебный проект!
участвует в фазе package
создаёт обычный jar
управляет MANIFEST.MF
НЕ включает зависимости
Результат:
	hw-1-10-1.0-SNAPSHOT.jar
java -cp target\hw-1-10-1.0-SNAPSHOT.jar ru.lesson.l10.Main			//запуск приложения
============= плагин maven-shade-plugin =========================
В реальных проектах, PRODUCTION, обычный jar почти не используют
тоже работает на фазе package
перепаковывает jar
добавляет все зависимости
может:
	заменить manifest
	перезаписать jar
	создать hw-1-10-1.0-SNAPSHOT-shaded.jar
java -jar target\hw-1-10-1.0-SNAPSHOT-shaded.jar				//запуск приложения
------------- если выбирать между maven-jar-plugin и maven-shade-plugin ----------------------
maven-shade-plugin
└── создаёт jar
└── пишет manifest
└── кладёт зависимости
! меньше точек отказа
